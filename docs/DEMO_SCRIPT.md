# Demo Script

## 2-3 Minute Flow

### 1. Open Homepage

Start on the homepage.

Say: "GridReady AI helps infrastructure teams evaluate whether a power-heavy project can get reliable power, at the right cost, on a workable timeline, before capital is committed."

Point out the demo-data notice so expectations are clear.

### 2. Explain The Problem

Say: "Data centers, manufacturers, EV charging networks, storage, and clean-energy projects face the same issue: land can move faster than power availability. Grid, queue timing, cooling, climate, carbon, and financial risk are usually fragmented across separate sources."

### 3. Use Location Analyzer

Scroll to Location Analyzer.

Select one market, for example:

- Dallas
- Northern Virginia
- Phoenix

Explain:

- The headline score is weighted Grid Readiness.
- Higher scores mean stronger readiness and lower risk.
- The strongest and weakest categories identify diligence priorities.
- Source confidence shows whether a score is using generated cache data, external sample data, or demo fallback.

### 4. Compare Markets

Scroll to Market Comparison.

Say: "This view supports early market shortlisting. The goal is not to make a final siting decision; it is to identify which markets deserve deeper diligence and which constraints need executive attention."

Toggle a market if useful and show how the ranking changes.

### 5. Run Scenario

Scroll to Scenario Simulator.

Choose:

```text
100 MW AI Data Center
```

Select Dallas or Northern Virginia.

Say: "The same market can score differently depending on project type. For data centers, Grid Access and Time-to-Power carry heavier implications because capacity delivery and queue timing can break the business case."

### 6. Explain Source Confidence

Point to the selected-case source lines.

Say: "The current public demo uses sample external-style data and local demo fallbacks. The architecture is ready for generated caches, but this is not yet live production data."

Labels:

- `Verified cache`: generated cache data
- `External sample`: checked-in sample records
- `Demo fallback`: local MVP estimate

### 7. Close

Say: "The next step is turning these source foundations into production data pipelines, moving from state-level samples to utility, county, subregion, queue-region, and site-level intelligence."
